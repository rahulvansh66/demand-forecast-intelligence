### drift_analysis.py - Statistical Distribution Comparison (Step 13)

**Purpose**: Rigorous statistical validation of training vs validation period consistency to ensure reliable model performance assessment.

**Core Functions:**

```python
def compare_temporal_distributions(train_data, validation_data, features)
```
- Kolmogorov-Smirnov tests for continuous features (sales distributions)
- Mann-Whitney U tests for median comparisons
- Chi-square tests for categorical feature distributions
- Effect size calculation and confidence intervals
- Multiple testing correction (Bonferroni/FDR)

```python
def analyze_seasonal_representativeness(train_data, validation_data, calendar_data)
```
- Validate 28-day validation period seasonal alignment
- Compare validation period to same-season historical periods
- Seasonal decomposition component comparison
- Holiday pattern consistency validation
- Weekend/weekday distribution consistency

```python
def detect_category_drift(sales_data, train_end_date, category_col='cat_id')
```
- Category-specific distribution shift detection
- Volume drift assessment by product category
- Price strategy evolution detection
- Promotional pattern changes between periods
- Category representation consistency validation

```python
def compute_drift_severity_scores(drift_test_results)
```
- Quantitative drift magnitude assessment
- Risk scoring for model performance impact
- Confidence interval calculation for drift estimates
- Business impact interpretation of statistical differences
- Drift threshold recommendations for model retraining

```python
def validate_temporal_split_integrity(sales_data, split_date)
```
- Temporal boundary validation (no overlap between train/test)
- Data leakage detection across temporal splits
- Time-series continuity validation
- Missing data pattern consistency between periods
